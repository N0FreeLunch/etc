서버에 접속을 하는 방법
- OS 마다 네트워크 포트 외부 컴퓨터와 통신할 수 있다.
- 80포트(HTTP), 443포트(HTTPS)
- 클라이언트에서 용청한 데이터가 들어가는데


리퀘스트 데이터 전달과정
- 클라이언트가 IP주소를 치면 : 아이피에 맞는 컴퓨터를 찾아서 연결해 준다.
- 컴퓨터실 30대의 컴퓨터가 하나의 공유기 물려있다.
- A 컴퓨터에서 B라는 컴퓨터를 찾아야 된다 : A => 공유기(30대) => B
- 공유기가 아이피를 30대의 컴퓨터에 할당한다.
- 30대 중에서 하나를 찾는 방식 : IP주소
- A의 리퀘스트가 B에 도착 B는 A의 정보(A의 IP주소)를 가진다.
- B는 OS API를 통해서 포트에 도착한 정보를 어떤 프로세스에 보내준다.
- 프로세스가 받은 정보를 처리를 해서 OS API를 통해서 A쪽으로 보낸다.
- B 리스폰스를 생성 => A의 IP주소로 보낸다.

정보를 처리하는 부분 (php, spring, nodejs express... 웹프레임워크)
포트에서 받은 정보를 프로세스에 전달하는 보조 도구들이 있다. (nginx, apache ... )


인터넷 (월드 와이드 웹)
- 원래는 서버의 IP주소로 접근
- 도메인 서버 (프록시 서버 : 중간 매개역할을 하는 서버)
- naver.com에 접속 : 클라이언트가 naver.com주소로 요청 => 인터넷 연결이 되어 있으면 기본적으로 연결되는 도메인 서버 IP가 있다.
- 도메인 서버(엄청 많음)는 naver.com이라는 서버 IP로 리퀘스트를 전달 해 준다.
- 리퀘스트 : 클라이언트 => 도메인서버 => 실제 웹 서비스 서버 (네이버 서버)
- 리스폰스 : 클라이언트 <= 도메인서버 <= 실제 웹 서비스 서버 (네이버 서버)
- 아이피 주소 갯수가 많지 않다. 도메인 서버는 공유기 역할을 함
- 도메인 서버에서 IP 갯수를 일부 할당 받는다. 유동아이피 계속 바꿔 줘서 클라이언트가 공용 아이피를 사용하게 하는 것
- 공용 아이를 여러 도메인 서버들이 나눠 갖는다. A 도메인 서버는 1~50번까지 써라, B 도메인 서버는 51~100번까지 써라

웰드 와이드 웹 : 전 세계적 단위로 IP를 연결하는 것 (공용 아이피)
예를 들어 인터넷 가입자수 5000만 룸, 아이피가 100만개 밖에 없다.
도메인서버 100개에 : 1만개씩 할 당, IP6 할당


룸 당 아이피 할당은 인터넷 서비스 연결하는 순간 : 브라우저 들어갈 때는 아니다. 공유기 켜서 KT, SKT, LGT 인증을 받았을 때 => 아이피 할당 받음

컴퓨터(192.168. 사설 아이피) - 공유기(도메인 서버에서 받는 공용 아이피)(IPV4) - 도메인서버 - 네이버서버

할당 받은 공용 아이피가 부족할 때
컴퓨터(192.168. 사설 아이피) - 공유기(도메인 서버에서 받는 공용 아이피)(IPV4 + IPV6) - 도메인서버 - 네이버서버

도메인 서버 = 네임서버
클라이언트 => (도메인서버 <=> 도메인서버 <=> 도메인서버 <=> 도메인서버 <=> 도메인서버) => 실제 웹 서비스 서버 (네이버 서버)
